# 预算编制向导式表单设计方案

## 📋 一、整体架构设计

### 1. Flowable 流程定义（BPMN）

在 BPMN 流程设计器中，需要配置以下节点：

```
开始事件 → 部门预算编制（用户任务） → 分公司初审（用户任务） → 
总部复审（用户任务） → 结束事件
```

**每个用户任务需要配置：**
- **任务监听器**：设置处理人/候选组
- **表单属性**：关联对应的表单 key
- **流转条件**：根据审批结果判断流向

## 📊 六、页面交互逻辑

### 向导式表单核心方法

```javascript
// 伪代码示例
export default {
  data() {
    return {
      activeStep: 0, // 当前步骤
      steps: [
        { title: '基本信息', component: 'BasicInfoForm' },
        { title: '收入类', component: 'IncomeDetailTable' },
        { title: '成本费用类', component: 'ExpenseDetailTable' },
        { title: '资产负债类', component: 'AssetDetailTable' },
        { title: '提交审核', component: 'SubmitConfirm' }
      ],
      formData: {
        basic: {},
        details: []
      }
    }
  },
  methods: {
    // 下一步
    handleNext() {
      if (this.validateCurrentStep()) {
        this.activeStep++
      }
    },
    // 上一步
    handlePrev() {
      this.activeStep--
    },
    // 保存草稿
    handleSaveDraft() {
      // 调用后端接口保存
      saveBudgetDraft(this.formData)
    },
    // 提交审核
    handleSubmit() {
      // 启动 Flowable 流程
      startProcess('budget_process_key', this.formData)
    }
  }
}
```

## 🎯 七、关键设计要点总结

### 1. 向导式设计
- 使用 Element UI Steps 组件，分步骤引导用户填写
- 每步独立验证，确保数据完整性
- 支持上一步/下一步导航

### 2. 动态表单
- 基于 `subject_type` 动态加载对应类型的科目
- 科目下拉框支持搜索过滤
- 只显示叶子节点科目（is_leaf=1）

### 3. 表格化录入
- 明细数据采用表格形式，支持增删改
- 实时计算合计数
- 支持批量导入/导出

### 4. 实时汇总
- 每步自动计算该类别合计数
- 最后一步展示总览和明细预览
- 提供数据校验提示

### 5. 审核简化
- 审核页面以只读展示为主
- 重点突出审批意见和操作按钮
- 支持查看流转记录和流程跟踪图

### 6. 流程集成
- 通过 `formKey` 关联 Flowable 流程定义
- 表单数据作为流程变量传递
- 支持流程回退、委派、转办等操作

### 7. 权限控制
- 根据用户角色显示不同的操作按钮
- 部门只能看到本部门数据
- 分公司可查看下属部门数据
- 总部可查看全公司数据

### 8. 数据一致性
- 遵循 RuoYi 框架字段规范
- create_by/update_by 使用 VARCHAR(64)，存储用户账号
- dept_id/org_id 使用 BIGINT(20)，关联 sys_dept.dept_id
- 时间字段使用 DATETIME

### 第一步：创建「向导式编制页面」

RuoYi 没有内置向导表单，但可以直接新建一个 Vue 页面，用 ElementUI `el-steps` 实现即可，完全兼容。

#### 1. 新建页面文件

在前端项目 `src/views/budget/` 下新建：

```
src/views/budget/
├─ index.vue           // 预算列表页（已生成）
├─ add.vue             // 向导式编制页（我们要建）
├─ detail.vue          // 查看/审核页（我们要建）
└─ steps/              // 步骤组件目录
   ├─ BasicInfo.vue    // 第1步：基本信息
   ├─ IncomeTable.vue  // 第2步：收入类
   ├─ ExpenseTable.vue // 第3步：成本费用类
   ├─ AssetTable.vue   // 第4步：资产负债类
   └─ SubmitConfirm.vue// 第5步：提交确认
```

#### 2. 向导页核心代码（add.vue）

```vue
<template>
  <div class="app-container">
    <el-card shadow="hover">
      <!-- 步骤条 -->
      <el-steps :active="activeStep" finish-status="success" align-center>
        <el-step title="基本信息" />
        <el-step title="收入类预算" />
        <el-step title="成本费用类" />
        <el-step title="资产负债类" />
        <el-step title="提交审核" />
      </el-steps>

      <!-- 步骤内容区 -->
      <div class="step-content mt-4">
        <!-- 第1步：基本信息 -->
        <basic-info v-if="activeStep === 0" ref="basicForm" :form="form.basic" />

        <!-- 第2步：收入类 -->
        <income-table v-if="activeStep === 1" ref="incomeForm" :list="form.incomeList" />

        <!-- 第3步：成本费用类 -->
        <expense-table v-if="activeStep === 2" ref="expenseForm" :list="form.expenseList" />

        <!-- 第4步：资产负债类 -->
        <asset-table v-if="activeStep === 3" ref="assetForm" :list="form.assetList" />

        <!-- 第5步：确认提交 -->
        <submit-confirm v-if="activeStep === 4" :form="form" />
      </div>

      <!-- 按钮栏 -->
      <div class="step-footer mt-4 text-right">
        <el-button @click="activeStep > 0 && activeStep--">上一步</el-button>
        <el-button type="primary" @click="nextStep" v-if="activeStep < 4">下一步</el-button>
        <el-button @click="saveDraft">保存草稿</el-button>
        <el-button type="success" @click="submitProcess" v-if="activeStep === 4">提交审核</el-button>
        <el-button @click="$router.go(-1)">取消</el-button>
      </div>
    </el-card>
  </div>
</template>

<script>
// 引入 5 个步骤组件
import BasicInfo from './steps/BasicInfo'
import IncomeTable from './steps/IncomeTable'
import ExpenseTable from './steps/ExpenseTable'
import AssetTable from './steps/AssetTable'
import SubmitConfirm from './steps/SubmitConfirm'

export default {
  name: 'BudgetAdd',
  components: { BasicInfo, IncomeTable, ExpenseTable, AssetTable, SubmitConfirm },
  data() {
    return {
      activeStep: 0,
      // 整个向导表单数据
      form: {
        basic: {},       // 基本信息
        incomeList: [],  // 收入明细
        expenseList: [], // 成本费用
        assetList: []    // 资产负债
      }
    }
  },
  methods: {
    // 下一步（带表单验证）
    async nextStep() {
      const valid = await this.$refs[`step${this.activeStep}Form`]?.validate()
      if (valid) this.activeStep++
    },
    // 保存草稿（调用后端接口）
    saveDraft() {
      this.$modal.confirm('确认保存草稿？').then(() => {
        this.$http.post('/budget/sheet/draft', this.form).then(res => {
          this.$msg.success('保存成功')
        })
      })
    },
    // 提交 Flowable 流程
    submitProcess() {
      this.$modal.confirm('确认提交审核？').then(() => {
        this.$http.post('/budget/sheet/submit', this.form).then(res => {
          this.$msg.success('提交成功')
          this.$router.go(-1)
        })
      })
    }
  }
}
</script>
```

### 第三步：Flowable 流程设计（最关键）

完全按设计的流程：**编制 → 分公司初审 → 总部复审 → 结束**

#### 2. 后端提交流程接口（Java）

```java
@PostMapping("/submit")
public AjaxResult submit(@RequestBody BudgetSheet form) {
    // 1. 保存预算主表+明细表
    budgetSheetService.save(form);
    
    // 2. 启动 Flowable 流程
    Map<String, Object> variables = new HashMap<>();
    variables.put("createBy", SecurityUtils.getUsername());
    variables.put("deptId", form.getOrgId());
    
    // 启动流程
    runtimeService.startProcessInstanceByKey(
        "budgetProcess",
        form.getId().toString(), 
        variables
    );
    
    return AjaxResult.success();
}
```
