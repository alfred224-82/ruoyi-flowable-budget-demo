## 三、设计规范遵循

### ✅ RuoYi框架规范
1. **字段命名**: 严格遵循数据库下划线命名 → Java驼峰命名
2. **人员字段**: `create_by`/`update_by` 使用 `String` 类型（VARCHAR(64)）
3. **部门字段**: `org_id`/`dept_id` 使用 `Long` 类型（BIGINT(20)）
4. **时间字段**: 使用 `Date` 类型（DATETIME）
5. **金额字段**: 使用 `BigDecimal` 类型（DECIMAL(18,2)）

### ✅ MyBatis-Plus规范
1. **实体类**: 使用 `@TableName` 指定表名
2. **主键**: 使用 `@TableId` 注解
3. **继承**: 主表继承 `BaseEntity`，明细表实现 `Serializable`

### ✅ BaseMapperPlus泛型规范
遵循 `<Mapper接口, Entity实体, VO视图对象>` 的三泛型定义

### ✅ 校验规范
1. **BO对象**: 添加完整的校验注解
2. **分组校验**: 区分新增（AddGroup）和修改（EditGroup）
3. **数值范围**: 月份1-12，金额>=0

## 五、注意事项

### ⚠️ 重要提醒
1. **数据库执行**: 先执行 `budget.sql` 建表脚本
2. **字段一致性**: 所有对象的字段必须与数据库表完全一致
3. **泛型顺序**: Mapper接口的泛型顺序不能错
4. **校验分组**: BO对象的校验要区分新增和修改场景
5. **Excel导出**: VO对象的 `@ExcelProperty` 配置要准确

### 🔧 常见问题
1. **编译错误**: 检查Mapper泛型的第三个参数是否为VO类
2. **字段映射错误**: 检查XML中的column是否与数据库字段名一致
3. **校验失败**: 检查BO对象的校验注解是否正确配置分组
