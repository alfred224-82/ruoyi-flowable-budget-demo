# 预算编制与审核功能测试指南

## 测试前准备

### 1. 数据库准备
确保以下表已创建：
- `budget_preparation` (预算编制主表)
- `budget_preparation_detail` (预算编制明细表)
- `budget_subject` (预算科目表)

SQL 脚本位置：`script/sql/mysql/` 目录

### 2. 权限配置
在系统中配置以下权限标识：
- `system:preparation:list` - 查询列表
- `system:preparation:query` - 查询详情
- `system:preparation:add` - 新增
- `system:preparation:edit` - 修改
- `system:preparation:remove` - 删除
- `system:preparation:submit` - 提交审核
- `system:preparation:approve` - 审批通过
- `system:preparation:reject` - 审批驳回
- `system:preparation:export` - 导出

### 3. 菜单配置
前端路由已在 `ruoyi-ui/src/router/index.js` 中配置：
- `/system/preparation` - 预算编制列表
- `/system/preparation/wizard` - 预算编制向导
- `/system/preparation/approval` - 预算审核列表
- `/system/preparation/approvalDetail` - 预算审核详情

## 异常场景测试

### 1. 重复提交审核
- 对已提交的记录再次提交，应该提示"状态不是草稿或驳回，不能提交审核"

### 2. 审批非待审核记录
- 对"草稿"状态的记录进行审批，应该提示"状态不是待审核，不能审批"

### 3. 驳回时不填理由
- 驳回时 reason 参数为空，应该提示"驳回时必须填写驳回意见"

### 4. 删除已通过记录
- 删除"Approved"状态的记录，应该提示"不是草稿或驳回状态，不能删除"

### 5. 并发提交
- 同一时刻多人提交同一预算单，应该有防重复提交机制

## 常见问题排查

### 1. Mapper 无法自动装配
这是正常的 IDE 警告，MyBatis 会在运行时生成实现类。只要项目能正常启动就没问题。

### 2. 权限不足
检查用户是否分配了相应的角色和权限标识。

### 3. 数据不一致
检查事务配置是否正确，关键操作是否有 `@Transactional` 注解。

### 4. 预算单号重复
检查 `generateSheetNo` 方法的并发安全性，必要时加锁。
