# 预算编制前端开发完成说明

## 一、已完成的前端文件

### 1. API接口文件
**文件路径**: `ruoyi-ui/src/api/system/preparation.js`

**包含的接口**:
- `listPreparation(query)` - 查询预算编制列表
- `getPreparation(id)` - 查询预算编制详细
- `addPreparation(data)` - 新增预算编制
- `updatePreparation(data)` - 修改预算编制
- `delPreparation(id)` - 删除预算编制
- `submitPreparation(id)` - 提交审核（单个）
- `batchSubmitPreparation(ids)` - 批量提交审核
- `listPreparationDetail(sheetId)` - 查询预算编制明细列表
- `savePreparationDetail(data)` - 保存预算编制明细
- `batchSavePreparationDetail(data)` - 批量保存预算编制明细
- `listFirstLevelSubject()` - 查询一级科目列表
- `listSecondLevelSubject(parentId)` - 查询二级科目列表

### 2. 预算编制查询页面
**文件路径**: `ruoyi-ui/src/views/system/preparation/index.vue`

**功能特性**:
- ✅ 查询条件：年度、月份、编制状态（草稿/审批中/已驳回/已通过）
- ✅ 表格显示：年月、编制状态、预算总额、预算单号、组织名称、操作
- ✅ 状态标签颜色区分：
  - 草稿：灰色 (info)
  - 审批中：橙色 (warning)
  - 已驳回：红色 (danger)
  - 已通过：绿色 (success)
- ✅ 操作按钮根据状态动态显示：
  - **草稿/已驳回状态**：修改、删除、提交审核
  - **审批中状态**：查看（只读）
- ✅ 批量操作：
  - 新建编制按钮
  - 批量提交审核按钮（仅当选中的记录都是草稿或驳回状态时可用）
- ✅ 多选框支持批量选择

### 3. 预算编制向导页面
**文件路径**: `ruoyi-ui/src/views/system/preparation/wizard.vue`

**向导步骤**:

#### 第一步：基础信息
- ✅ 预算年度：自动填写今年，不可修改
- ✅ 预算月份：可选择（1-12月），默认当前月
- ✅ 编制人：自动填充当前登录用户，不可修改
- ✅ 组织名称：自动填充当前用户所属部门，不可修改
- ✅ 备注：可选填写
- ✅ 点击"下一步"后自动保存主表数据到 `budget_preparation` 表

#### 第二步：科目预算编制
- ✅ 按一级科目分多个 Tab 页
- ✅ 每个 Tab 页显示对应的二级科目列表
- ✅ 二级科目可填写预算金额（使用数字输入框，最小值为0）
- ✅ 实时计算并显示预算总额
- ✅ 懒加载：切换 Tab 时才加载对应的二级科目数据
- ✅ 此步骤没有"上一步"按钮（防止误操作导致数据丢失）

#### 第三步：完成编制
- ✅ 展示基础信息汇总（预算单号、期间、组织、编制人等）
- ✅ 展示预算总额
- ✅ 展示所有填写的科目预算明细表格
- ✅ 只有"完成编制"按钮，没有"下一步"
- ✅ 点击完成后批量保存明细数据到 `budget_preparation_detail` 表

**通用功能**:
- ✅ 每一步都有"返回"按钮，可返回查询页面
- ✅ 支持编辑模式：从查询页面点击"修改"进入，加载已有数据
- ✅ 支持查看模式：审批中的记录只能查看，不能编辑
- ✅ 表单验证：基础信息必填项验证
- ✅ 数据校验：第二步至少填写一个科目的预算金额

## 二、后端需要开发的接口

### 1. 预算编制主表接口（BudgetPreparationController）

#### 基础 CRUD 接口
```java
// GET /system/preparation/list
// 查询预算编制列表

@GetMapping("/list")
```

## 六、技术要点说明

### 1. 前端技术栈
- Vue 2.x
- Element UI
- Vuex（获取用户信息）
- Vue Router

### 2. 关键实现细节

#### 年度月份处理
- 年度使用 `el-date-picker` type="year"，格式为 "yyyy"
- 月份使用 `el-select`，值为数字 1-12
- 显示时组合成 "YYYY-MM" 格式

#### 科目数据结构
```javascript
// 一级科目
{
  id: 1,
  subjectCode: "1001",
  subjectName: "营业收入",
  level: 1
}

// 二级科目（带预算金额）
{
  id: 2,
  subjectCode: "1001001",
  subjectName: "主营业务收入",
  parentId: 1,
  budgetAmount: 0,  // 用户填写
  remark: ""        // 用户填写
}
```

#### 数据提交流程
1. 第一步：保存主表数据，获取 `sheetId`
2. 第二步：用户填写各科目的预算金额
3. 第三步：汇总所有明细，批量保存到 `budget_preparation_detail` 表
4. 更新主表的 `total_budget` 字段

#### 编辑模式处理
- 通过路由参数 `?id=xxx` 判断是否为编辑模式
- 通过路由参数 `?viewOnly=true` 判断是否为只读模式
- 编辑模式加载已有数据并填充到表单
- 只读模式直接跳到第三步展示数据
