## 一、验收总览

## 八、回归测试清单

每次版本更新后必须重新验证以下核心功能：

### 8.1 核心流程（必测）

- [ ] 新建预算编制（向导3步完整流程）
- [ ] 编辑预算编制（草稿状态，直接进入第2步）
- [ ] 删除预算编制（草稿状态）
- [ ] 完成编制（Draft/Rejected → Completed，向导第3步确认）
- [ ] 提交审核（仅 Completed，状态变为 Pending_Review + approval_stage=Dept）
- [ ] 批量提交审核
- [ ] 部门经理审批通过（approval_stage 变为 Branch）
- [ ] 分公司本部审批通过（approval_stage 变为 HQ）
- [ ] 总公司本部审批通过（status 变为 Approved，预算锁定）
- [ ] 批量审批通过
- [ ] 审批驳回（必填意见校验，status 变为 Rejected）
- [ ] 三级驳回统一回退（部门/分公司/总公司驳回都回退到编制人员）
- [ ] 驳回后重新编辑（状态重置为 Draft，需重新完成编制）
- [ ] 驳回后完成编制再提交（Draft → Completed → Pending_Review）
- [ ] 明细查询
- [ ] 明细保存（单条）
- [ ] 明细批量保存
- [ ] 差异金额和差异率自动计算

### 8.2 权限验证（必测）

- [ ] 总公司查看所有部门数据
- [ ] 分公司查看下属部门数据
- [ ] 部门仅查看本部门数据
- [ ] 数据脱敏规则生效
- [ ] 无权限操作返回403

### 8.3 边界场景（必测）

- [ ] 不填写任何科目金额时的提示
- [ ] 金额为负数时前端限制
- [ ] 超大金额精度处理
- [ ] 批量操作事务回滚
