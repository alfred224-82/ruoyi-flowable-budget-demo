#### 1.3 业务逻辑说明

##### 1.3.1 预算单号生成规则
- 格式: `BG-YYYYMM-XXX`
- 示例: `BG-202606-001`
- 自动生成逻辑: 根据年月统计已有单据数量，序号从 001 开始递增

##### 1.3.2 状态流转
- `Draft` (草稿): 初始状态，可编辑、删除、提交
- `Pending_Review` (待审核): 提交后状态，可审批通过或驳回
- `Approved` (已通过): 审批通过后的最终状态
- `Rejected` (已驳回): 审批驳回后可重新编辑再提交

##### 1.3.3 查询条件
- 年度 (`budgetYear`): 精确匹配
- 月份 (`budgetMonth`): 精确匹配 (1-12)
- 状态 (`status`): 精确匹配
- 组织ID (`orgId`): 用于部门权限控制
- 预算单号 (`sheetNo`): 模糊查询

##### 1.3.4 删除校验
- 只有 `Draft` 和 `Rejected` 状态的预算编制可以删除
- 其他状态删除时会抛出异常提示

##### 1.3.5 提交审核校验
- 只有 `Draft` 和 `Rejected` 状态可以提交审核
- 提交后状态变更为 `Pending_Review`
- 清空驳回信息

##### 1.3.6 审批通过
- 只有 `Pending_Review` 状态可以审批
- 审批通过后状态变更为 `Approved`

##### 1.3.7 审批驳回
- 只有 `Pending_Review` 状态可以驳回
- **驳回时必须填写驳回意见**（强制校验）
- 驳回后状态变更为 `Rejected`
- 记录驳回来源为 `HQ`（总部）
- 设置当前处理人为操作人

  - 路径: `ruoyi-system/src/main/java/com/ruoyi/system/service/IBudgetPreparationDetailService.java`
  
- **Service 实现**: `BudgetPreparationDetailServiceImpl.java`
  - 路径: `ruoyi-system/src/main/java/com/ruoyi/system/service/impl/BudgetPreparationDetailServiceImpl.java`
  
- **Mapper**: `BudgetPreparationDetailMapper.java` (已存在)
  - 路径: `ruoyi-system/src/main/java/com/ruoyi/system/mapper/BudgetPreparationDetailMapper.java`

#### 2.2 API 接口列表

| 接口路径 | 方法 | 功能 | 权限标识 |
|---------|------|------|---------|
| `/system/preparation/detail/list/{sheetId}` | GET | 查询预算编制明细列表 | system:preparation:query |
| `/system/preparation/detail` | POST | 保存单条明细（新增或修改） | system:preparation:edit |
| `/system/preparation/detail/batch` | POST | 批量保存明细 | system:preparation:edit |

#### 2.3 业务逻辑说明

##### 2.3.1 自动计算差异
- **差异金额** = 预算金额 - 实际金额
- **差异率** = (差异金额 / 预算金额) × 100%
- 当预算金额为 0 时，差异率为 0

##### 2.3.2 批量保存策略
- 先删除该预算单的所有旧明细数据
- 再批量插入新的明细数据
- 保证数据一致性（事务控制）

##### 2.3.3 排序规则
- 明细列表按 `sortOrder` 升序排列

## 技术特性

### 1. 事务控制
- 所有写操作（新增、修改、删除、提交、审批）都使用 `@Transactional` 注解
- 确保数据一致性

### 2. 权限控制
- 使用 `@SaCheckPermission` 注解进行权限校验
- 每个接口都有对应的权限标识

### 3. 日志记录
- 使用 `@Log` 注解记录操作日志
- 包含操作标题和业务类型

### 4. 防重复提交
- 关键接口使用 `@RepeatSubmit` 注解防止重复提交

### 5. 参数校验
- 使用 JSR-303 注解进行参数校验
- 分组校验：`AddGroup`（新增）、`EditGroup`（修改）

### 6. 异常处理
- 使用 `ServiceException` 抛出业务异常
- 统一的异常提示信息

## 数据库表结构

### budget_preparation (预算编制主表)
主要字段：
- `id`: 主键ID
- `sheet_no`: 预算单号（唯一）
- `org_id`: 组织ID
- `org_name`: 组织名称
- `budget_year`: 预算年度
- `budget_month`: 预算月份
- `budget_period`: 预算期间（YYYY-MM）
- `status`: 状态
- `reject_level`: 驳回来源
- `reject_reason`: 驳回理由
- `current_handler`: 当前处理人
- `total_budget`: 预算总额
- `process_instance_id`: Flowable流程实例ID

### budget_preparation_detail (预算编制明细表)
主要字段：
- `id`: 主键ID
- `sheet_id`: 预算单ID（外键）
- `subject_code`: 科目编码
- `subject_name`: 科目名称
- `subject_type`: 科目类型
- `budget_amount`: 预算金额
- `actual_amount`: 实际金额
- `variance_amount`: 差异金额
- `variance_rate`: 差异率
- `dept_id`: 部门ID
- `dept_name`: 部门名称
- `sort_order`: 排序号

## 注意事项

### 1. 编译警告
IDE 可能会显示 Mapper 无法自动装配的警告，这是正常的，因为 MyBatis 会在运行时动态生成 Mapper 的实现类。项目已配置全局的 `@MapperScan` 扫描。

### 2. 待完善功能
以下功能已在代码中标记为 TODO，需要根据实际业务需求完善：

1. **提交审核时的明细校验** (`BudgetPreparationServiceImpl.submitForReview`)
   - 检查是否有明细数据
   - 检查明细金额是否合理
   - 检查必填科目是否完整

2. **校验结果查询** (`BudgetPreparationServiceImpl.getValidationResults`)
   - 返回详细的校验信息
   - 包括校验不通过的科目列表

3. **保存前的数据校验** (`validEntityBeforeSave` 方法)
   - 科目编码是否存在
   - 科目类型是否正确
   - 金额范围校验

4. **预算总额自动汇总**
   - 明细保存后自动更新主表的 total_budget 字段

5. **Flowable 流程集成**
   - 提交审核时启动 Flowable 流程
   - 审批时调用 Flowable 的任务完成接口
   - 流程结束后更新预算编制状态

### 3. 部门权限控制
前端 approval.vue 页面实现了基于角色的部门过滤：
- 总公司角色：可查看所有部门
- 分公司角色：只能查看分公司下部门
- 部门领导：只能查看自己部门

后端查询接口已支持 `orgId` 参数，前端传入即可实现权限控制。

### 4. 批量审批请求对象
Controller 中定义了内部类 `BatchApproveRequest` 用于接收批量审批请求：
```java
{
  "ids": [1, 2, 3],
  "remark": "审批意见"
}
```
